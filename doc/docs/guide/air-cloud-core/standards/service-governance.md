# 服务治理标准

服务治理标准用于抽象服务注册、服务发现、键值配置中心等能力。它解决的是“服务在哪里、配置在哪里、公共运行信息在哪里”的问题。

## 标准接口

| Standard | 作用 |
| --- | --- |
| `IServerCenterStandard` | 服务注册、注销、服务发现等服务中心能力 |
| `IKVCenterStandard` | 键值对中心能力，例如读取远程配置、写入元数据 |
| `IConsulServiceOptionsConfigureStandard` | Consul 模块自身的配置构建标准 |

## 当前实现

| 模块 | 实现内容 |
| --- | --- |
| `Air.Cloud.Modules.Consul` | 基于 Consul 实现服务中心、KV 中心和远程配置加载 |

## 使用建议

- 如果服务只需要本地配置，不要强行接入服务治理模块。
- 如果使用 Consul 作为配置中心，要同时关注本地启动配置和远程配置加载结果。
- 服务注册与配置加载属于启动链路能力，调用时机必须早于业务服务初始化。
